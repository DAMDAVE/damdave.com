The process of making websites and hosting them on the internet is quite a bit complex but I'll expain it by dividing it into three basic sections.

The first stage is where a programmer like myself writes the code that defines the website in his local computer

The second is where he uploads the code to a repository hosting service such as ;

  • GitHub
  • GitLab
  • Netlify
  • BitBucket and so on . . .

These services take the codes on the local computer and store it on the cloud where it can easily be accessed for runtime on the server.

With both stages one and two running, the content of the website can be fully uploaded on the internet. However, the third and final stage is a very important and necessary one when a programmer is dealing with a real life situation website. This is where we buy the domain names.
Now, what is a domain name, it isn't what you haven't seen before. The names of websites ending with .com, .net, .org e.t.c are domains e.g netnaija.com, 9jarocks.com and the likes. You wouldn't want your website to look so long and weird like some of the links in the previous section so that's why a domain name is necessary.
